저장소 체크아웃 안에서 `pnpm ragit init`은 기본적으로 7단계 인터랙티브 흐름으로 동작합니다.

1. 환경 검사 (`git init` 선택 포함)
2. 초기화 모드 확인
3. 루트 `AGENTS.md` 로드/생성
4. 문서 템플릿 범위 확인 (ADR/PRD/SRS/SPEC/Plan/DDD/Glossary/PBD)
5. `.ragit/guide` 증분 생성
6. zvec 저장소 bootstrap
7. 요약 및 다음 액션 출력

`SPEC` 템플릿은 구현체 상세 기능 명세에, `PBD` 템플릿은 위상 구조와 결속 아키텍처 설명에 사용됩니다.

## `init`이 준비하는 것

- `.ragit/` 기준의 Git 인지 control plane 준비
- 루트 `AGENTS.md`의 로드 또는 생성 흐름
- `.ragit/config.toml`
- `.ragit/guide/templates/*`
- `.ragit/guide/guide-index.json`
- `.ragit/store/meta.json`
- `documents`, `chunks` 빈 zvec 컬렉션

## `init`이 준비하지 않는 것

- searchable corpus 생성 없음
- chunk 레코드 또는 manifest 생성 없음
- 저장소 문서 스캔 또는 chunk 생성 없음
- 빈 zvec 컬렉션에 대한 실제 벡터 upsert 없음

즉, `init`은 저장소를 **control-plane ready** 및 **zvec-store ready** 상태로 만들지만, **search-ready** 상태로 만들지는 않습니다.
검색 가능한 지식 상태는 `pnpm ragit ingest ...` 이후에 형성됩니다.

`storage.backend = "zvec"`는 이제 canonical backend를 의미하며, `init`은 저장소 문서를 인덱싱하지 않은 빈 zvec 저장소만 bootstrap합니다.

## 대표 사용자 시나리오

### U1. 신규 로컬 부트스트랩

현재 디렉터리가 Git 저장소가 아니고 `AGENTS.md`와 `.ragit/`도 아직 없을 때의 흐름입니다.

1. `pnpm ragit init` 실행
2. Git 부재 감지
3. 대화형에서 `git init` 제안
4. `AGENTS.md` 생성
5. `.ragit/config.toml`, `.ragit/guide/templates/*`, `guide-index.json`, 빈 zvec 저장소 생성
6. 다음 액션 출력: `ragit hooks install`, `ragit ingest --all`

핵심은 초기화는 끝나지만, 아직 검색 가능한 지식 인덱스는 만들어지지 않는다는 점입니다.

### U2. 기존 Git 저장소 + 기존 `AGENTS.md`

이미 Git 저장소이며 루트 `AGENTS.md`가 존재할 때의 흐름입니다.

1. `pnpm ragit init` 실행
2. `AGENTS.md`를 읽기 전용으로 로드
3. 누락된 `.ragit` control-plane 파일만 생성하고 필요 시 zvec 저장소 bootstrap
4. `guide-index.json` 재생성

핵심은 기존 지침문은 수정되지 않고, 표준 가이드 구조만 증분 보강된다는 점입니다.

### U3. 부분 초기화된 저장소 재실행

`.ragit/`은 이미 있지만 일부 템플릿 또는 인덱스만 빠져 있을 때의 흐름입니다.

1. `pnpm ragit init` 실행
2. 기존 파일 유지
3. 누락 템플릿만 생성
4. `guide-index.json` 최신화

핵심은 `init`이 idempotent/incremental하게 동작하여 안전하게 재실행 가능하다는 점입니다.

### U4. 비대화형 또는 CI 초기화

TTY가 없거나 자동화 환경에서 기본값으로 실행해야 할 때의 흐름입니다.

1. `pnpm ragit init --yes` 실행
2. Git 저장소면 바로 진행
3. 비Git이면 `--git-init` 없을 때 실패, 있으면 자동 `git init`
4. 동일한 guide + zvec-store 결과를 table 또는 JSON summary로 반환

핵심은 CI에서도 guide/control-plane 준비와 빈 canonical store 준비까지는 가능하지만, 실제 인덱싱은 이후 `ingest` 단계가 담당한다는 점입니다.

## 비대화형 실행

```bash
pnpm ragit init --yes
pnpm ragit init --yes --git-init
pnpm ragit init --yes --output json
```

## 생성 구조

```text
.ragit/
  config.toml
  guide/
    guide-index.json
    templates/
  store/
    meta.json
    documents/
    chunks/
```
