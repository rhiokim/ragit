# init



무엇을 하는 명령인가 [#무엇을-하는-명령인가]

`init`은 discover-first bootstrap 명령입니다. 저장소 루트로 정규화한 뒤 Git 문맥을 확인하고, 저장소를 스캔하고, `empty`, `existing`, `docs-heavy`, `monorepo` 중 하나로 분류하며, 문서 coverage/maturity를 평가하고, 누락된 기초 문서를 계획한 뒤 control plane과 로컬 zvec store를 부팅합니다.

언제 쓰는가 / 언제 쓰지 않는가 [#언제-쓰는가--언제-쓰지-않는가]

언제 쓰는가 [#언제-쓰는가]

* 저장소에서 RAGit을 처음 설정할 때
* 기존 문서를 우선 재사용하고 부족한 초안만 생성하고 싶을 때
* 에이전트나 CI가 읽을 수 있는 initialization report가 필요할 때

언제 쓰지 않는가 [#언제-쓰지-않는가]

* searchable knowledge만 필요할 때. init 이후 <a href="/ko/docs/commands/ingest">`ingest`</a>를 실행하십시오.
* 현재 상태 점검만 필요할 때. <a href="/ko/docs/commands/status">`status`</a> 또는 <a href="/ko/docs/commands/doctor">`doctor`</a>를 사용하십시오.

기본 구문 [#기본-구문]

```bash
pnpm ragit init [--cwd <path>] [--yes|--non-interactive] [--mode auto|empty|existing|monorepo|docs-heavy] [--strategy minimal|balanced|full] [--dry-run] [--merge-existing] [--git-init] [--output text|json|both]
```

인자와 옵션 [#인자와-옵션]

* `--cwd <path>`: 다른 저장소 경로를 대상으로 실행합니다. Git worktree 내부 경로를 주면 저장소 루트 기준으로 정규화됩니다.
* `--yes`: 기본값을 수락하고 질문을 건너뜁니다.
* `--non-interactive`: `--yes`와 같은 의도로 비대화형 실행을 강제합니다.
* `--mode auto|empty|existing|monorepo|docs-heavy`: 저장소 모드 자동 판정을 덮어씁니다. 기본값은 `auto`입니다.
* `--strategy minimal|balanced|full`: 1단계 기초 문서 생성 강도를 제어합니다. 기본값은 `balanced`입니다.
* `--dry-run`: 파일을 쓰지 않고 전체 리포트만 계산합니다.
* `--merge-existing`: 기존 문서를 1차 소스로 우선 재사용합니다. 기본 동작입니다.
* `--git-init`: 비대화형 모드에서 git 저장소를 자동 생성합니다.
* `--output text|json|both`: 사람이 읽는 요약, JSON 요약, 둘 다 중 하나를 선택합니다.

입력/출력 계약 [#입력출력-계약]

* 이 명령은 플래그 입력만 지원합니다. `--input <path|->` payload 모드는 없습니다.
* JSON 출력에는 다음 키가 포함됩니다.
  * `executionMode`
  * `repositoryMode`
  * `strategy`
  * `scan`
  * `coverage`
  * `maturity`
  * `knowledgeMap`
  * `actions`
  * `bootstrap`
  * `nextActions`
* `actions`는 1단계 기초 문서에 대한 create/reuse/skip 계획을 보고합니다.
* `bootstrap`은 실제 AGENTS/guide/zvec 상태 또는 `--dry-run`일 때 계획 상태를 보고합니다.

대표 사용 예시 [#대표-사용-예시]

사람용 터미널 예시:

```bash
pnpm ragit init --mode auto --strategy balanced --merge-existing
```

에이전트 호출 예시:

```bash
pnpm ragit init --yes --git-init --output json
```

미리보기 예시:

```bash
pnpm ragit init --dry-run --output json
```

실패/주의 사항 [#실패주의-사항]

* 비대화형 모드에서 git 저장소가 아니고 `--git-init`도 없으면 실패합니다.
* TTY가 없는 환경에서는 대화형 모드를 사용할 수 없습니다.
* `--dry-run`은 저장소 분석은 수행하지만 파일과 zvec store는 만들지 않습니다.
* zvec bootstrap은 현재 `darwin/arm64`, `linux/arm64`, `linux/x64`만 지원합니다.
* `init`은 저장소 문서를 읽고 분류할 수 있지만, 저장소를 search-ready 상태로 만들지는 않습니다. 초기화 후 <a href="/ko/docs/commands/ingest">`ingest`</a>를 실행하십시오.
