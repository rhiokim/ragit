# init



무엇을 하는 명령인가 [#무엇을-하는-명령인가]

`init`은 저장소를 RAGit에 맞게 준비합니다. git 컨텍스트를 확인하고, RAGit 구조를 만들고, `AGENTS.md`를 로드 또는 생성하며, 가이드 파일과 zvec 저장소 레이아웃을 초기화합니다.

언제 쓰는가 / 언제 쓰지 않는가 [#언제-쓰는가--언제-쓰지-않는가]

언제 쓰는가 [#언제-쓰는가]

* 저장소에서 RAGit을 처음 설정할 때
* 표준 `.ragit/**` 구조와 가이드 골격이 필요할 때
* 레거시 JSON store 마이그레이션이 필요한지 확인하고 싶을 때

언제 쓰지 않는가 [#언제-쓰지-않는가]

* 이미 초기화된 저장소에서 새 문서만 인덱싱하고 싶을 때. <a href="/ko/docs/commands/ingest">`ingest`</a>를 사용하십시오.
* 현재 상태만 점검하고 싶을 때. <a href="/ko/docs/commands/status">`status`</a> 또는 <a href="/ko/docs/commands/doctor">`doctor`</a>를 사용하십시오.

기본 구문 [#기본-구문]

```bash
pnpm ragit init [--cwd <path>] [--yes|--non-interactive] [--git-init] [--output text|json|both]
```

인자와 옵션 [#인자와-옵션]

* `--cwd <path>`: 다른 저장소 경로를 대상으로 실행합니다.
* `--yes`: 기본값을 수락하고 질문을 건너뜁니다.
* `--non-interactive`: `--yes`와 같은 의도로 비대화형 실행을 강제합니다.
* `--git-init`: 비대화형 모드에서 git 저장소를 자동 생성합니다.
* `--output text|json|both`: 사람이 읽는 표, JSON 요약, 둘 다 중 하나를 선택합니다.

입력/출력 계약 [#입력출력-계약]

* 이 명령은 플래그 입력만 지원합니다. `--input <path|->` payload 모드는 없습니다.
* JSON 출력에는 `mode`, `git`, `agents`, `guide`, `storage`, `nextActions`가 포함됩니다.
* 에이전트가 마이그레이션 필요 여부를 판단해야 한다면 `--output json`이 적합합니다.

대표 사용 예시 [#대표-사용-예시]

사람용 터미널 예시:

```bash
pnpm ragit init
```

에이전트 호출 예시:

```bash
pnpm ragit init --yes --git-init --output json
```

실패/주의 사항 [#실패주의-사항]

* 비대화형 모드에서 git 저장소가 아니고 `--git-init`도 없으면 실패합니다.
* TTY가 없는 환경에서는 대화형 모드를 사용할 수 없습니다.
* `init`은 저장소를 준비할 뿐이고 문서를 인덱싱하지는 않습니다. 초기화 후 <a href="/ko/docs/commands/ingest">`ingest`</a>를 실행하십시오.
