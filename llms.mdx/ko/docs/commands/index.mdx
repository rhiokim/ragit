# Commands



`commands` 그룹은 개별 명령 상세 문서의 새 진입점입니다.
각 leaf 페이지는 한 개의 실행 가능한 CLI 명령을 사용자 관점과 에이전트 연동 관점에서 함께 설명합니다.

여기서 시작하십시오 [#여기서-시작하십시오]

* 저장소를 처음 설정할 때는 <a href="/ko/docs/commands/init">`init`</a>부터 읽으십시오.
* 인덱싱된 지식에서 바로 검색할 때는 <a href="/ko/docs/commands/query">`query`</a>를 읽으십시오.
* 다음 에이전트 단계에 넣을 압축된 패킷이 필요하면 <a href="/ko/docs/commands/context/pack">`context pack`</a>을 읽으십시오.
* 일반 검색이 아니라 현재 작업을 이어서 복원하려면 <a href="/ko/docs/commands/memory/recall">`memory recall`</a>을 읽으십시오.

명령어 패밀리 [#명령어-패밀리]

* 최상위 명령: <a href="/ko/docs/commands/init">`init`</a>, <a href="/ko/docs/commands/describe">`describe`</a>, <a href="/ko/docs/commands/ingest">`ingest`</a>, <a href="/ko/docs/commands/query">`query`</a>, <a href="/ko/docs/commands/status">`status`</a>, <a href="/ko/docs/commands/doctor">`doctor`</a>
* Config: <a href="/ko/docs/commands/config">`config`</a>
* Context: <a href="/ko/docs/commands/context">`context`</a>
* Hooks: <a href="/ko/docs/commands/hooks">`hooks`</a>
* Memory: <a href="/ko/docs/commands/memory">`memory`</a>
* Migrate: <a href="/ko/docs/commands/migrate">`migrate`</a>

이 문서들을 읽는 방법 [#이-문서들을-읽는-방법]

* 모든 leaf 페이지는 동일한 섹션 구조를 사용하므로 구문, 옵션, payload 예시, 주의점을 빠르게 찾을 수 있습니다.
* `--input <path|->`를 지원하는 명령은 JSON payload 예시를 포함합니다.
* `--view`를 지원하는 명령은 `minimal`, `default`, `full` 차이를 설명합니다.
* `--dry-run`을 지원하는 명령은 실제 변경 전에 무엇을 검증하는지 설명합니다.

빠른 매핑 [#빠른-매핑]

* `query`: "이 질문에 맞는 인덱싱된 지식은 무엇인가?"
* `context pack`: "이 목표를 위해 최소-충분 컨텍스트 패킷은 무엇인가?"
* `memory wrap`: "작업을 멈추기 전에 무엇을 기록해야 하는가?"
* `memory recall`: "작업을 다시 시작하기 전에 무엇을 불러와야 하는가?"
* `memory promote`: "어떤 내용을 지속 가능한 프로젝트 기억으로 승격해야 하는가?"

예전 진입점이 더 익숙하시다면, 기존 <a href="/ko/docs/cli-reference">CLI Reference</a> 페이지는 이제 이 그룹으로 안내하는 호환 페이지 역할만 합니다.
